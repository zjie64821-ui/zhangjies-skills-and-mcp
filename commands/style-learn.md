# Style Learn: 写作风格学习引擎

从指定的文件库或 Zotero 集合中深度学习写作风格，生成风格档案供后续写作使用。

## 触发场景
- 用户需要为某个写作项目建立风格基准
- 用户切换到新的研究领域，需要适应不同的学术写作范式
- 用户说"学习一下这个文件夹的风格"、"看看这些文章怎么写的"

## 工作流

### 第一步：确定学习来源

用户通过 $ARGUMENTS 指定来源，支持以下格式：
- 本地文件夹路径：如 `/Users/zhangjie/Desktop/某项目/参考文献/`
- Zotero 集合名称：如 `能源转型政策写作`
- 指定文件列表：如 `file1.pdf, file2.pdf`

如果参数为空，询问用户提供范文来源。

### 第二步：读取范文

**本地文件夹**：
1. 用 Glob 扫描目录下所有文本类文件（`*.pdf`, `*.txt`, `*.md`, `*.docx`）
2. 逐篇读取内容（PDF 用 pdf skill 或直接 Read）
3. 如果文件超过 20 篇，随机选取 15-20 篇代表性样本（覆盖不同作者、不同子主题）

**Zotero 集合**：
1. 用 `mcp__zotero-mcp__search_collections` 找到目标集合
2. 用 `mcp__zotero-mcp__get_collection_items` 获取条目列表
3. 用 `mcp__zotero-mcp__get_content` 提取全文内容
4. 优先选择有 PDF 附件的条目

### 第三步：深度风格分析

对每篇范文执行以下分析，然后综合汇总：

#### A. 句式结构
- 平均句长（中文按字符数，英文按单词数）
- 长短句分布比例
- 复合句 vs 简单句的使用频率
- 被动语态使用频率（英文）
- "的"字结构使用频率（中文）
- 倒装、设问等修辞手法

#### B. 词汇偏好
- 高频学术用语列表（出现 3 次以上的领域术语）
- 过渡词/连接词偏好（如"进一步地"vs"此外"vs"与此同时"）
- 表述习惯（如"本研究"vs"本文"vs"我们"；"结果表明"vs"研究发现"）
- 避免/少用的词汇（对比 AI 默认输出习惯）
- 中英文混合写作时的语码切换模式

#### C. 段落组织
- 段落平均长度
- 段落内部结构（主题句→论据→小结 / 总分总 / 演绎式）
- 段落间衔接方式
- 列举/枚举的使用模式

#### D. 论证结构（学术论文特有）
- 研究背景的写法模式（宏观→微观→研究缺口→本文贡献）
- 文献综述的组织方式（按主题/按时间/按方法）
- 方法论描述的详细程度
- 结果呈现的叙事方式
- 讨论部分的论证策略

#### E. 格式规范
- 引用格式（脚注/尾注/行内引用）
- 图表标注方式
- 标题层级和编号规则
- 缩写使用习惯

#### F. "去AI味"特征（关键）
- AI 常用但范文不用的词汇/短语列表
- AI 常用但范文不用的句式结构
- 范文中独特的"人味"表达（不完美的、口语化的、带有个人判断的表述）
- 数值和数据的呈现方式（AI 倾向于过度精确或过度模糊）

### 第四步：生成风格档案

将分析结果写入风格档案文件：

**保存位置**：当前项目目录下的 `.claude/style-profile.md`
如果不在项目目录中，保存到 `~/writing/style-profiles/{项目名或来源名}.md`

**风格档案模板**：

```markdown
# 写作风格档案

## 元信息
- 来源：[文件夹路径或 Zotero 集合名]
- 分析篇数：[N] 篇
- 语言：中文 / 英文 / 中英混合
- 学科领域：[如：供应链管理/能源政策]
- 生成时间：[日期]

## 核心风格特征（摘要，200字以内）
[一段话概括这个范文库最显著的写作特征]

## 句式规则
- [规则1：如"偏好15-25字的中短句，每段不超过5句"]
- [规则2]
- ...

## 词汇白名单（范文偏好用词）
| 范文用词 | 替代的AI常用词 |
|---------|--------------|
| [词1] | [AI词1] |
| ... | ... |

## 词汇黑名单（AI味重、范文不用）
- [词1]
- [词2]
- ...

## 过渡与衔接模板
- 引出新话题：[示例]
- 补充论据：[示例]
- 转折对比：[示例]
- 总结归纳：[示例]

## 段落模板
- 研究背景段：[结构描述]
- 文献综述段：[结构描述]
- 方法论段：[结构描述]
- ...

## 去AI味检查清单
- [ ] 是否使用了黑名单词汇？
- [ ] 句式是否过于整齐对仗？（AI特征）
- [ ] 过渡是否过于丝滑无缝？（AI特征）
- [ ] 是否缺少具体数据和案例支撑？
- [ ] 语气是否过于笃定？（缺乏学术审慎）

## 写作示例（从范文中提取的代表性段落）
### 中文示例
[粘贴2-3段最能代表风格的段落]

### 英文示例
[粘贴2-3段最能代表风格的段落]
```

### 第五步：确认与迭代

1. 向用户展示风格档案的核心发现
2. 询问用户：
   - "这个风格分析是否符合你的预期？"
   - "有没有需要调整或补充的地方？"
   - "是否要把某些范文的权重提高？"
3. 根据反馈修正风格档案

## 注意事项
- 分析时要区分"领域术语"和"风格特征"，前者是内容，后者是表达
- 中文和英文要分别分析，不能混为一谈
- 风格档案应该具体到可以直接指导写作，避免泛泛而谈
- 如果范文之间存在明显风格差异（不同期刊/不同作者），在档案中标注变体

$ARGUMENTS
